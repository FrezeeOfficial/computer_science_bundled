BEST AUTHENTICATION WAY KNOWN:

    when a user first uses the web app, ask them to get a generated key from the software. this key will then be strongly encrytped with details that allow for tracing if leaked. it will be unique to user and
    stored as a uuid.

    users can reject keys and the key must be linked to only one device ip. 

    the key when decrypted will contain data like the devices screen size, components web browser and things which can be compared to the web page requester. If data is considerably mismatched to the user then
    the key will be auto revoked.

    data will also be compared to database aswell and also the clients components

    the spec below shows a method which is fall proof, data cannot be tampered and if attempted. Will result in immediate key rejection. The token acts as a token which links the user to app in its decrypted form
